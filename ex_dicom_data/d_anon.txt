# The format of this file is simple:

#
# The <optional_new_value> can be used as follows:
# a) blank or ...   -> ignored, the tag is cleared
# b) <LUT_XX>       -> will replace the tag value based on a look-up table, eg: <LUT_MyTRIAL> will look for a file
#                      called AnonymizationLUT_MyTRIAL.txt and will look up the new value for the tag there.
#                      See the example LUT file in the application bundle for how to format this file.
#                      Just as this configuration file, the LUT file, can be either (searched in this order:)
#                      1- In the Application Bundle itself (allowing you to distribute a copy of the app with a predefined config)
#                      2- On the Desktop
#                      3- In the folder where the Application Bundle is (for example to place a copy of the app in a standard output folder)
#                      4- In ~/Library/Applications Support/DICOManonymizer (for general use)
#                      Note that look up values are treated as string values.
#                      Note that anonymizing using a LUT takes much longer because it requires all files to be parsed before writing
#                      Note that the current version of this application supports *one* tag to be replaced by a LUT value.
# c) anything else  -> will replace the tag with 'anything else'.
